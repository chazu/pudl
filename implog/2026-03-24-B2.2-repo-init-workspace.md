# B2.2 -- Extend `pudl repo init` to Generate workspace.cue

**Date:** 2026-03-24
**Bead:** B2.2
**Epic:** E2 -- Per-Repo Workspace Support

## Summary

Extended `pudl repo init` to create a complete per-repo workspace layout. After init, `.pudl/` contains `workspace.cue`, `schema/`, and `definitions/` directories.

## Changes

### `internal/repo/init.go`
- After creating `.pudl/`, generates `workspace.cue` with the directory name as the workspace name
- Creates `schema/` and `definitions/` subdirectories under `.pudl/`
- Adds `.gitkeep` files to empty directories so git tracks them
- Respects `--force` flag: without force, existing `workspace.cue` is preserved; with force, it is regenerated
- Added `generateWorkspaceCue(name string) string` helper function

### `internal/repo/init_test.go`
- `TestInit_CreatesWorkspaceCue` -- verifies workspace.cue exists and contains directory name
- `TestInit_CreatesSubdirectories` -- verifies schema/ and definitions/ exist with .gitkeep files
- `TestInit_Force_OverwritesWorkspaceCue` -- verifies --force regenerates workspace.cue
- `TestInit_NoForce_PreservesWorkspaceCue` -- verifies workspace.cue is not overwritten without --force
- `TestInit_WorkspaceCue_ValidCUE` -- parses generated workspace.cue with CUE compiler, confirms validity

## Public API

No new public API. Existing `repo.Init(opts InitOptions)` now produces additional files in the `.pudl/` directory.

## Resulting Layout

```
my-project/
  .pudl/
    workspace.cue          # name: "my-project"
    schema/
      .gitkeep
    definitions/
      .gitkeep
  .claude/
    skills/
      pudl-core.md         # (existing behavior)
```
