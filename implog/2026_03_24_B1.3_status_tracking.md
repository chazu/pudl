# B1.3 -- Convergence Status Tracking

## Summary
Added a `status` column to the catalog database that tracks convergence state throughout the ACUTE cycle. Each definition entry now carries a queryable status value.

## Changes

### internal/database/catalog_migrations.go
- Added `ensureStatusColumn()` migration (idempotent, checks column existence via `pragma_table_info`)
- Column defaults to `'unknown'` for existing rows

### internal/database/catalog.go
- Added `Status *string` field to `CatalogEntry` struct
- Updated `AddEntry` to default status to `"unknown"` when nil
- Updated all SELECT queries and Scan operations to include the `status` column (GetEntry, GetEntryByProquint, QueryEntries, GetCollectionItems, GetCollectionByID)
- Registered `ensureStatusColumn()` in `createTables`

### internal/database/catalog_observe.go
- Updated all SELECT/Scan operations (GetLatestObserve, GetLatestObserveByOrigin, GetLatestObserveByContentHash)

### internal/database/catalog_manifest.go
- Updated all SELECT/Scan operations (GetManifestActions, GetLatestManifestAction)

### internal/database/catalog_artifacts.go
- Updated all SELECT/Scan operations (GetLatestArtifact, GetLatestArtifactByOrigin, SearchArtifacts)

### internal/database/catalog_identity.go
- Updated all SELECT/Scan operations (FindByContentHash, FindByResourceID)

### internal/database/catalog_status.go (new)
- `DefinitionStatus` struct: Definition, Status, UpdatedAt, DiffCount
- `UpdateStatus(definitionName, status)` -- validates against known status values, updates latest entry
- `GetDefinitionStatuses()` -- returns per-definition status summary

### internal/drift/checker.go
- After computing drift result, calls `db.UpdateStatus(defName, "clean"|"drifted")`

### cmd/export_actions.go
- Added `markConverging()` helper that opens the catalog DB and sets status to `"converging"` for non-clean definitions after export
- Called from both `runExportOne` and `runExportAll`

### internal/mubridge/manifest.go
- After ingesting each manifest action, sets status to `"converged"` (exit code 0) or `"failed"` (non-zero exit code)

## Public API

```go
// database package
type DefinitionStatus struct {
    Definition string
    Status     string
    UpdatedAt  time.Time
    DiffCount  int
}

func (c *CatalogDB) UpdateStatus(definitionName string, status string) error
func (c *CatalogDB) GetDefinitionStatuses() ([]DefinitionStatus, error)
```

## Status State Machine
```
unknown -> clean|drifted       (drift check)
drifted -> converging          (export-actions)
converging -> converged|failed (ingest-manifest)
converged|failed -> clean|drifted (drift check)
```

## Tests

- `TestUpdateStatus_Valid` -- sets each valid status, verifies persistence
- `TestUpdateStatus_Invalid` -- rejects invalid status values
