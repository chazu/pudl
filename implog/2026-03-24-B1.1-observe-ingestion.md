# B1.1 — Observe Result Ingestion

**Date:** 2026-03-24
**Bead:** B1.1 (Epic 1: Close the ACUTE Feedback Loop)

## Summary

Implemented `pudl ingest-observe` command that accepts `mu observe --json` NDJSON output and stores observed live state in the catalog, making it available as the "live" side of drift detection.

## Files Created

- `internal/mubridge/ingest.go` — `IngestObserveResults` function: NDJSON parsing, content-hash dedup, raw file storage, catalog entry creation with `entry_type = "observe"`
- `cmd/ingest_observe.go` — CLI command `pudl ingest-observe` with `--path` and `--origin` flags
- `internal/database/catalog_observe.go` — `GetLatestObserve(definitionName)` and `GetLatestObserveByContentHash(definitionName, contentHash)` query methods
- `internal/database/catalog_observe_test.go` — Tests for `GetLatestObserve`, `GetLatestObserve_NoResults`, `GetLatestObserve_DoesNotReturnArtifacts`, `GetLatestObserveByContentHash`
- `internal/mubridge/ingest_test.go` — Tests for `IngestObserveResults_Basic`, `_Dedup`, `_EmptyInput`, `_InvalidJSON`, `_TargetWithoutPrefix`, `_CustomOrigin`

## Files Modified

- `internal/drift/checker.go` — Prefer observe results over artifacts for live state resolution
- `internal/drift/checker_test.go` — Added `TestChecker_PrefersObserveOverArtifact`
