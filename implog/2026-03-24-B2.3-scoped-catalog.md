# B2.3 -- Workspace-Scoped Catalog Queries

**Date:** 2026-03-24
**Bead:** B2.3

## Summary

Made catalog queries workspace-aware. When running inside a per-repo workspace, `pudl list`, `pudl import`, and `pudl drift check` now default to scoping by the workspace origin.

## Changes

### cmd/root.go
- Added `PersistentPreRunE` to root command that runs `workspace.NewContext()` on every invocation
- Exposed package-level `wsCtx *workspace.Context` for use by subcommands

### cmd/list.go
- Added `--all-workspaces` flag to bypass workspace origin filtering
- When inside a workspace and `--origin` is not explicitly set, `list` defaults the origin filter to `wsCtx.EffectiveOrigin` (the workspace name)

### cmd/import.go
- Single-file and batch imports auto-set origin to `wsCtx.EffectiveOrigin` when inside a workspace and `--origin` is not explicitly provided

### cmd/drift_check.go
- `initDriftChecker()` now calls `checker.SetOriginFilter(wsCtx.EffectiveOrigin)` when inside a workspace

### internal/drift/checker.go
- Added `originFilter string` field to `Checker` struct
- Added `SetOriginFilter(origin string)` method
- `Check()` uses origin-filtered DB queries (`GetLatestObserveByOrigin`, `GetLatestArtifactByOrigin`) when `originFilter` is non-empty; falls back to unfiltered queries otherwise

### internal/database/catalog_observe.go
- Added `GetLatestObserveByOrigin(definitionName, origin string)` -- same as `GetLatestObserve` with `AND origin = ?`

### internal/database/catalog_artifacts.go
- Added `GetLatestArtifactByOrigin(definition, method, origin string)` -- same as `GetLatestArtifact` with `AND origin = ?`

## Public API

- `drift.Checker.SetOriginFilter(origin string)` -- restricts catalog queries to a specific origin
- `database.CatalogDB.GetLatestObserveByOrigin(definitionName, origin string) (*CatalogEntry, error)`
- `database.CatalogDB.GetLatestArtifactByOrigin(definition, method, origin string) (*CatalogEntry, error)`
- CLI flag: `pudl list --all-workspaces`
