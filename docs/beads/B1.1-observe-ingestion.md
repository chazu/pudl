# B1.1 — Observe Result Ingestion

**Epic:** E1 — Close the ACUTE Feedback Loop
**Blocked by:** nothing
**Blocks:** B1.3 (status tracking)

## Summary

Add a `pudl ingest-observe` command that accepts `mu observe --json` output and stores observed live state in the catalog, making it available as the "live" side of drift detection.

## Background

- mu's observe protocol emits NDJSON to stdout, one JSON object per target
- Each object matches the `pudl/mu.#ObserveResult` CUE schema (defined in `pudl/mu/mu.cue`)
- The schema has identity field `target` and tracked fields `state`, `diff`
- The catalog `entry_type` field currently accepts `"import"` and `"artifact"`
- Drift checker (`internal/drift/checker.go`) currently reads live state from the latest artifact entry

## Implementation

### 1. New file: `internal/mubridge/ingest.go`

Create an `IngestObserveResults` function:

```go
package mubridge

// ObserveInput represents a single observe result from mu.
type ObserveInput struct {
    Target string                 `json:"target"`
    State  map[string]interface{} `json:"state"`
    Diff   map[string]interface{} `json:"diff,omitempty"`
}

// IngestObserveResults processes mu observe output and stores results in the catalog.
// Returns the number of results ingested and any error.
func IngestObserveResults(db *database.CatalogDB, reader io.Reader, origin string) (int, error)
```

Logic:
- Read NDJSON from `reader` line by line
- For each line, unmarshal into `ObserveInput`
- Extract target name (strip leading `//` if present) -- this maps to the definition name
- Serialize the `state` map as JSON, compute SHA256 content hash
- Check for dedup: if content hash matches latest observe entry for this definition, skip
- Create a `database.CatalogEntry` with:
  - `EntryType`: `"observe"` (new value)
  - `Origin`: `origin` (default `"mu-observe"`)
  - `Schema`: `"pudl/mu.#ObserveResult"`
  - `Definition`: target name (mapped to definition)
  - `Format`: `"json"`
  - Content hash and resource ID computed from target name + schema
- Store raw JSON in `~/.pudl/data/raw/YYYY/MM/DD/` (reuse existing storage pattern)
- Insert into catalog via `db.AddEntry()`

### 2. New file: `cmd/ingest_observe.go`

```go
var ingestObserveCmd = &cobra.Command{
    Use:   "ingest-observe",
    Short: "Ingest mu observe results into the catalog",
    Long: `Read mu observe output (NDJSON) and store observed state in the catalog.
The observed state becomes the "live" side for subsequent drift checks.

Examples:
    mu observe --json | pudl ingest-observe
    pudl ingest-observe --path observe-results.json`,
    RunE: func(cmd *cobra.Command, args []string) error { ... },
}
```

Flags:
- `--path <file>` — read from file instead of stdin
- `--origin <string>` — override origin (default: `"mu-observe"`)

Logic:
- If `--path` is set, open file; otherwise use `os.Stdin`
- Call `mubridge.IngestObserveResults(db, reader, origin)`
- Print summary: "Ingested N observe results"

### 3. Modify: `internal/database/catalog.go`

Add a new query method:

```go
// GetLatestObserve returns the most recent observe entry for a definition.
func (db *CatalogDB) GetLatestObserve(definitionName string) (*CatalogEntry, error)
```

SQL: `SELECT ... FROM catalog_entries WHERE entry_type = 'observe' AND definition = ? ORDER BY import_timestamp DESC LIMIT 1`

No schema migration needed — `entry_type` is a TEXT column, adding `"observe"` as a value requires no DDL change.

### 4. Modify: `internal/drift/checker.go`

In the `Check` method, change live state resolution order:

```go
// Try observe results first (most recent actual state from mu)
entry, err := c.db.GetLatestObserve(opts.DefinitionName)
if err != nil || entry == nil {
    // Fall back to latest artifact
    entry, err = c.db.GetLatestArtifact(opts.DefinitionName, "")
}
```

This is a ~5-line change in `checker.go` around line 45.

## Acceptance Criteria

- `echo '{"target":"//my_app","state":{"replicas":3}}' | pudl ingest-observe` creates a catalog entry
- `pudl list --schema pudl/mu.#ObserveResult` shows the ingested entry
- `pudl drift check my_app` uses the observe result as live state (not the last import)
- Content-hash deduplication prevents duplicate observe entries
