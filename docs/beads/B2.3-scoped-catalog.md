# B2.3 — Workspace-Scoped Catalog Queries

**Epic:** E2 — Per-Repo Workspace Support
**Blocked by:** B2.1 (workspace discovery)
**Blocks:** nothing

## Summary

Make catalog queries workspace-aware by defaulting the `origin` filter to the current workspace name. Add `--all-workspaces` flag to bypass this filter.

## Background

- `CatalogEntry` already has an `origin` field (TEXT, nullable)
- `QueryEntries()` already supports filtering by origin
- Currently origin is set to the source file path or user-provided value
- After this change, imports within a workspace automatically tag entries with the workspace name as origin
- Queries default to filtering by workspace origin

## Implementation

### 1. Modify: `cmd/root.go` — Inject workspace context

Add workspace discovery to the root command's `PersistentPreRunE`:

```go
import "pudl/internal/workspace"

var wsCtx *workspace.Context

var rootCmd = &cobra.Command{
    PersistentPreRunE: func(cmd *cobra.Command, args []string) error {
        // Discover workspace (may be nil for global mode)
        var err error
        wsCtx, err = workspace.NewContext()
        if err != nil {
            return fmt.Errorf("workspace discovery: %w", err)
        }
        return nil
    },
}
```

Expose `wsCtx` for use by subcommands. Use a package-level variable or pass via Cobra's context.

### 2. Modify: `cmd/list.go` — Default origin filter

Add flag:

```go
var allWorkspaces bool

func init() {
    listCmd.Flags().BoolVar(&allWorkspaces, "all-workspaces", false,
        "Show entries from all workspaces (default: current workspace only)")
}
```

In the query logic, when `wsCtx.Workspace != nil && !allWorkspaces`, set the origin filter:

```go
if wsCtx != nil && wsCtx.Workspace != nil && !allWorkspaces {
    // Default to current workspace
    queryOpts.Origin = wsCtx.EffectiveOrigin
}
```

### 3. Modify: `cmd/import.go` — Tag entries with workspace origin

When importing within a workspace, set the origin to workspace name unless explicitly overridden:

```go
if wsCtx != nil && wsCtx.Workspace != nil && origin == "" {
    origin = wsCtx.EffectiveOrigin
}
```

This ensures all imports within a workspace are automatically scoped.

### 4. Modify: `cmd/drift_check.go` — Workspace-scoped drift

When in a workspace, `drift check --all` should only check definitions from the workspace's definitions directory (handled by B2.4). The catalog queries for live state should also be workspace-scoped:

```go
// In initDriftChecker, pass workspace origin to checker
checker := drift.NewChecker(defDisc, db, cfg.DataPath)
if wsCtx != nil && wsCtx.Workspace != nil {
    checker.SetOriginFilter(wsCtx.EffectiveOrigin)
}
```

### 5. Modify: `internal/drift/checker.go` — Optional origin filter

Add an origin filter field:

```go
type Checker struct {
    defDisc      *definition.Discoverer
    db           *database.CatalogDB
    dataPath     string
    originFilter string // empty means no filter
}

func (c *Checker) SetOriginFilter(origin string) {
    c.originFilter = origin
}
```

Use it when querying for live state:

```go
// In Check(), when looking up latest observe/artifact
if c.originFilter != "" {
    entry, err = c.db.GetLatestObserveByOrigin(opts.DefinitionName, c.originFilter)
} else {
    entry, err = c.db.GetLatestObserve(opts.DefinitionName)
}
```

### 6. Modify: `internal/database/catalog.go` — Origin-filtered queries

Add origin-aware variants:

```go
func (db *CatalogDB) GetLatestObserveByOrigin(definitionName, origin string) (*CatalogEntry, error)
func (db *CatalogDB) GetLatestArtifactByOrigin(definitionName, method, origin string) (*CatalogEntry, error)
```

These add `AND origin = ?` to the WHERE clause.

## Acceptance Criteria

- `pudl list` in a workspace shows only that workspace's entries
- `pudl list --all-workspaces` shows everything
- `pudl import` in a workspace tags entries with the workspace name
- `pudl drift check` in a workspace uses workspace-scoped live state
- Outside any workspace, behavior is unchanged (no origin filter applied)
