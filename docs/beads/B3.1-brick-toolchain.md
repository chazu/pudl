# B3.1 — BRICK-Aware Export Actions

**Epic:** E3 — BRICK-Aware Toolchain Mapping
**Blocked by:** nothing
**Blocks:** nothing

## Summary

When a definition uses a `brick.#Target` schema, extract the `toolchain` field from the declared state and use it directly in mu.json output, bypassing the prefix heuristic.

## Background

- `internal/mubridge/export.go` resolves toolchains via `resolveToolchain()`, which matches schema ref prefixes
- `DriftInput` carries `SchemaRef` and `Sources` but no explicit toolchain
- `brick.#Target` (defined in `pudl/brick/brick.cue`) has a required `toolchain` field
- When a definition is `brick.#Target &`, the declared keys in the drift result will contain `toolchain`

## Implementation

### 1. Modify: `internal/mubridge/export.go`

Add a `BrickToolchain` field to `DriftInput`:

```go
type DriftInput struct {
    Result         *drift.DriftResult
    SchemaRef      string
    Sources        []string
    BrickToolchain string // explicit toolchain from BRICK metadata (takes precedence)
}
```

Update `ExportMuConfig` to check this field first:

```go
for _, input := range results {
    if input.Result.Status == "clean" {
        continue
    }

    // BRICK toolchain takes precedence over prefix heuristic
    var toolchain string
    if input.BrickToolchain != "" {
        toolchain = input.BrickToolchain
    } else {
        toolchain = resolveToolchain(input.SchemaRef, mappings)
    }

    toolchainsSeen[toolchain] = true
    // ... rest unchanged
}
```

### 2. Modify: `cmd/export_actions.go`

When building `DriftInput`, detect BRICK targets and extract toolchain:

```go
func buildDriftInput(def *definition.DefinitionInfo, report *drift.DriftResult) *mubridge.DriftInput {
    input := &mubridge.DriftInput{
        Result:    report,
        SchemaRef: def.SchemaRef,
        Sources:   []string{def.FilePath},
    }

    // If this is a BRICK target, extract toolchain from declared state
    if isBrickTarget(def.SchemaRef) {
        if tc, ok := report.DeclaredKeys["toolchain"]; ok {
            if tcStr, ok := tc.(string); ok {
                input.BrickToolchain = tcStr
            }
        }
    }

    return input
}

// isBrickTarget checks if a schema ref is a BRICK target type.
func isBrickTarget(schemaRef string) bool {
    return strings.Contains(schemaRef, "brick.#Target") ||
           strings.Contains(schemaRef, "brick.#Kit") ||
           strings.Contains(schemaRef, "brick.#Interface")
}
```

Update `runExportOne` and `runExportAll` to use `buildDriftInput`.

### 3. Also extract BRICK `config` field

For BRICK targets, the mu.json target config should come from the `config` sub-field, not the top-level declared keys (which include BRICK metadata like `kind`, `implements`, etc.):

```go
if isBrickTarget(def.SchemaRef) {
    // Use config sub-field as mu target config
    if configMap, ok := report.DeclaredKeys["config"]; ok {
        if cm, ok := configMap.(map[string]interface{}); ok {
            // Override: use BRICK config, not all declared keys
            input.BrickConfig = cm
        }
    }
}
```

Add `BrickConfig` field to `DriftInput` and use it in `ExportMuConfig`:

```go
// In ExportMuConfig, when building target config:
config := make(map[string]any)
if input.BrickConfig != nil {
    for k, v := range input.BrickConfig {
        config[k] = v
    }
} else {
    for k, v := range input.Result.DeclaredKeys {
        config[k] = v
    }
}
```

## Acceptance Criteria

- BRICK `#Target` definitions use their `toolchain` field directly in mu.json
- BRICK `config` sub-field (not all declared keys) becomes the mu target config
- Non-BRICK definitions are unaffected (prefix heuristic still works)
- `BrickToolchain` takes absolute precedence over prefix matching
