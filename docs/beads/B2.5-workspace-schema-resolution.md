# B2.5 — Workspace-Aware Schema Resolution

**Epic:** E2 — Per-Repo Workspace Support
**Blocked by:** B2.1 (workspace discovery)
**Blocks:** nothing

## Summary

Extend schema inference and validation to search per-repo `.pudl/schema/` before global `~/.pudl/schema/` before embedded bootstrap schemas. Per-repo schemas shadow global schemas with the same fully-qualified name.

## Background

- Schema inference (`internal/inference/`) discovers candidate schemas by walking the schema directory
- Validation (`internal/validator/`) loads CUE schemas from the schema path
- Both currently accept a single schema path from config
- The workspace context (B2.1) provides `SchemaSearchPaths` — an ordered list
- CUE's module system can handle cross-directory imports if modules reference each other

## Implementation

### 1. Modify: `internal/inference/inference.go`

The schema inferrer needs to collect candidates from multiple directories.

Find where candidates are loaded (likely in `SelectCandidates` or the schema loading function) and update to accept a path list:

```go
// Before: single path
func (i *Inferrer) loadSchemas(schemaPath string) ([]SchemaCandidate, error)

// After: multiple paths, first-found wins for name conflicts
func (i *Inferrer) loadSchemas(schemaPaths []string) ([]SchemaCandidate, error) {
    seen := make(map[string]bool) // canonical name -> already loaded
    var candidates []SchemaCandidate

    for _, sp := range schemaPaths {
        localCandidates, err := i.loadSchemasFromDir(sp)
        if err != nil {
            continue // skip inaccessible dirs
        }
        for _, c := range localCandidates {
            if !seen[c.Name] {
                seen[c.Name] = true
                candidates = append(candidates, c)
            }
        }
    }

    return candidates, nil
}
```

Update the `Inferrer` constructor to accept multiple paths:

```go
type Inferrer struct {
    schemaPaths []string
    // ...
}

func NewInferrer(schemaPaths ...string) *Inferrer {
    return &Inferrer{schemaPaths: schemaPaths}
}
```

### 3. Modify: `internal/importer/importer.go`

The importer creates the inferrer. Update it to pass through multiple schema paths:

```go
// In ImportFileWithFriendlyIDs or wherever the inferrer is created:
inferrer := inference.NewInferrer(schemaPaths...)
```

The importer's constructor needs to accept the path list:

```go
type EnhancedImporter struct {
    schemaPaths []string
    db          *database.CatalogDB
    // ...
}
```

### 4. Modify: `cmd/import.go`

When constructing the importer, pass workspace-aware schema paths:

```go
var schemaPaths []string
if wsCtx != nil && wsCtx.Workspace != nil {
    schemaPaths = wsCtx.SchemaSearchPaths
} else {
    schemaPaths = []string{cfg.SchemaPath}
}
importer := importer.NewEnhancedImporter(schemaPaths, db, ...)
```

### 5. Modify: `internal/schema/` — Schema discovery

If there's a schema listing/browsing function, update it similarly:

```go
func ListSchemas(schemaPaths []string) ([]SchemaInfo, error)
```

### 6. CUE Module Resolution

For per-repo schemas that import global schemas, the per-repo `.pudl/schema/cue.mod/module.cue` can declare a dependency:

```cue
module: "workspace.schemas@v0"

deps: {
    "pudl.schemas@v0": {
        v: "v0.0.0"
        default: true
    }
}
```

This is optional — many per-repo schemas will be self-contained. But document this as a pattern for advanced use.

## Acceptance Criteria

- Schema inference finds schemas from per-repo `.pudl/schema/` first
- Per-repo schemas shadow global schemas with the same name
- Validation works against per-repo schemas
- Imports within a workspace use the per-repo schema set
- When no per-repo schema matches, global schemas are still used
- Backward compatible: single-path constructors still work
