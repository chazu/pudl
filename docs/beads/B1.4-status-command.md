# B1.4 — Status CLI Command

**Epic:** E1 — Close the ACUTE Feedback Loop
**Blocked by:** B1.3 (status tracking)
**Blocks:** nothing

## Summary

Add a `pudl status` command that shows per-definition convergence state in a readable table.

## Implementation

### 1. New file: `cmd/status.go`

```go
var statusCmd = &cobra.Command{
    Use:   "status [definition]",
    Short: "Show convergence status of definitions",
    Long: `Display the current convergence status of all definitions or a specific one.

Status values:
  unknown     — no drift check has been run
  clean       — drift check found no differences
  drifted     — drift check found differences
  converging  — actions exported to mu, awaiting result
  converged   — mu build succeeded
  failed      — mu build failed

Examples:
    pudl status
    pudl status my_app
    pudl status --json`,
    Args: cobra.MaximumNArgs(1),
    RunE: func(cmd *cobra.Command, args []string) error { ... },
}
```

Flags:
- `--json` — output as JSON array instead of table

### 2. Default output (table format)

```
Definition          Status       Last Updated
──────────────────  ───────────  ────────────────────
api_server          converged    2026-03-24T10:15:00Z
monitoring          drifted      2026-03-24T10:12:00Z  (3 differences)
config_file         clean        2026-03-24T10:14:00Z
database            failed       2026-03-24T10:15:00Z
worker              unknown      —
```

Logic:
- Open database
- If a definition name is given, show detailed status for that one definition (include latest drift differences if drifted)
- If no name given, call `db.GetDefinitionStatuses()` and display table
- For drifted definitions, optionally show difference count from latest drift report

### 3. JSON output

```json
[
  {"definition": "api_server", "status": "converged", "updated_at": "2026-03-24T10:15:00Z"},
  {"definition": "monitoring", "status": "drifted", "updated_at": "2026-03-24T10:12:00Z", "diff_count": 3}
]
```

### 4. Single definition detail

```
$ pudl status monitoring

Definition: monitoring
Status:     drifted
Last Check: 2026-03-24T10:12:00Z

Differences (3):
  ~ alert_threshold: 80 -> 90
  + new_channel: #ops
  - deprecated_webhook: https://...

Last Convergence: 2026-03-24T09:00:00Z (converged)
```

For single-definition view:
- Load latest drift report from `drift.ReportStore`
- Load latest manifest-action entry from catalog
- Combine into a unified view

2. **TestStatusCommand_MixedStatuses** — definitions with various statuses, verify correct table output
3. **TestStatusCommand_SingleDefinition** — verify detailed single-definition output includes drift differences

## Acceptance Criteria

- `pudl status` shows a table of all definitions with their convergence state
- `pudl status my_app` shows detailed info for a single definition
- `pudl status --json` outputs machine-readable JSON
- Status values are human-readable and color-coded in terminal (green=clean/converged, yellow=drifted/converging, red=failed)
