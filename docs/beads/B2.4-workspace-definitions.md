# B2.4 — Workspace-Aware Definition Discovery

**Epic:** E2 — Per-Repo Workspace Support
**Blocked by:** B2.1 (workspace discovery)
**Blocks:** nothing

## Summary

Extend the definition discoverer to search multiple directories: per-repo `.pudl/definitions/` first, then global `~/.pudl/schema/definitions/`. This makes definitions portable across repos while preserving backward compatibility.

## Background

- `definition.Discoverer` takes a single `schemaPath` and looks in `<schemaPath>/definitions/`
- It needs to accept a list of paths and search them in order
- Per-repo definitions shadow global definitions with the same name (first match wins)
- The workspace context (B2.1) provides `DefinitionSearchPaths`

## Implementation

### 1. Modify: `internal/definition/discovery.go`

Change `Discoverer` to accept multiple paths:

```go
type Discoverer struct {
    searchPaths []string // ordered list of schema paths to search
}

// NewDiscoverer creates a discoverer with a single schema path (backward compatible).
func NewDiscoverer(schemaPath string) *Discoverer {
    return &Discoverer{searchPaths: []string{schemaPath}}
}

// NewMultiDiscoverer creates a discoverer that searches multiple paths in order.
func NewMultiDiscoverer(schemaPaths []string) *Discoverer {
    return &Discoverer{searchPaths: schemaPaths}
}
```

Update `definitionsDir()` to return multiple dirs:

```go
func (d *Discoverer) definitionsDirs() []string {
    var dirs []string
    for _, sp := range d.searchPaths {
        dirs = append(dirs, filepath.Join(sp, "definitions"))
    }
    return dirs
}
```

Update `ListDefinitions()`:

```go
func (d *Discoverer) ListDefinitions() ([]DefinitionInfo, error) {
    seen := make(map[string]bool)
    var definitions []DefinitionInfo

    for _, defsDir := range d.definitionsDirs() {
        if _, err := os.Stat(defsDir); os.IsNotExist(err) {
            continue
        }

        err := filepath.Walk(defsDir, func(path string, info os.FileInfo, err error) error {
            if err != nil || info.IsDir() || !strings.HasSuffix(info.Name(), ".cue") {
                return nil
            }

            fileDefs, err := d.parseDefinitionsFromFile(path)
            if err != nil {
                return nil
            }

            for _, def := range fileDefs {
                if !seen[def.Name] {
                    seen[def.Name] = true
                    definitions = append(definitions, def)
                }
            }
            return nil
        })
        if err != nil {
            return nil, fmt.Errorf("failed to walk definitions directory %s: %w", defsDir, err)
        }
    }

    sort.Slice(definitions, func(i, j int) bool {
        return definitions[i].Name < definitions[j].Name
    })

    return definitions, nil
}
```

The `seen` map ensures per-repo definitions shadow global ones.

### 2. Modify: `cmd/` commands that create Discoverers

Update all commands that create a `definition.Discoverer` to use workspace-aware paths when available:

Commands to update:
- `cmd/definition.go` (list, show, validate, graph)
- `cmd/drift_check.go` (drift check)
- `cmd/export_actions.go` (export-actions)
- `cmd/repo.go` (repo validate)

Pattern for each:

```go
// Before:
defDisc := definition.NewDiscoverer(cfg.SchemaPath)

// After:
var defDisc *definition.Discoverer
if wsCtx != nil && wsCtx.Workspace != nil {
    defDisc = definition.NewMultiDiscoverer(wsCtx.SchemaSearchPaths)
} else {
    defDisc = definition.NewDiscoverer(cfg.SchemaPath)
}
```

To avoid repeating this in every command, add a helper to `cmd/root.go`:

```go
func newDefinitionDiscoverer() (*definition.Discoverer, error) {
    cfg, err := config.Load()
    if err != nil {
        return nil, err
    }
    if wsCtx != nil && wsCtx.Workspace != nil {
        return definition.NewMultiDiscoverer(wsCtx.SchemaSearchPaths), nil
    }
    return definition.NewDiscoverer(cfg.SchemaPath), nil
}
```
