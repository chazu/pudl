# B2.2 — Extend `pudl repo init` to Generate workspace.cue

**Epic:** E2 — Per-Repo Workspace Support
**Blocked by:** B2.1 (workspace discovery)
**Blocks:** nothing

## Summary

Update `pudl repo init` to create a `workspace.cue` file inside `.pudl/`, along with `schema/` and `definitions/` subdirectories. This makes `pudl repo init` the entry point for per-repo workspace setup.

## Background

- `internal/repo/init.go` currently creates `.pudl/` and installs Claude skills
- It does not create `workspace.cue`, `schema/`, or `definitions/` directories
- After this change, `pudl repo init` produces a complete per-repo workspace

## Implementation

### 1. Modify: `internal/repo/init.go`

After creating `.pudl/` directory (line 42), add:

```go
// Create workspace.cue
workspaceCuePath := filepath.Join(pudlDir, "workspace.cue")
if _, err := os.Stat(workspaceCuePath); os.IsNotExist(err) || opts.Force {
    dirName := filepath.Base(dir)
    content := generateWorkspaceCue(dirName)
    if err := os.WriteFile(workspaceCuePath, []byte(content), 0644); err != nil {
        return fmt.Errorf("creating workspace.cue: %w", err)
    }
    if opts.Verbose {
        fmt.Printf("Created %s\n", workspaceCuePath)
    }
}

// Create schema directory
schemaDir := filepath.Join(pudlDir, "schema")
if err := os.MkdirAll(schemaDir, 0755); err != nil {
    return fmt.Errorf("creating schema/: %w", err)
}

// Create definitions directory
defsDir := filepath.Join(pudlDir, "definitions")
if err := os.MkdirAll(defsDir, 0755); err != nil {
    return fmt.Errorf("creating definitions/: %w", err)
}

// Create .gitkeep in empty directories so git tracks them
for _, d := range []string{schemaDir, defsDir} {
    gitkeep := filepath.Join(d, ".gitkeep")
    if _, err := os.Stat(gitkeep); os.IsNotExist(err) {
        os.WriteFile(gitkeep, []byte(""), 0644)
    }
}
```

Add helper function:

```go
func generateWorkspaceCue(name string) string {
    return fmt.Sprintf(`// PUDL workspace configuration.
// This file marks the root of a per-repo PUDL workspace.

// Workspace name — used as the origin for catalog entries.
// Must be unique across all workspaces sharing the same ~/.pudl/ catalog.
name: %q

// Optional: override toolchain mappings for this workspace.
// These take priority over global config and built-in defaults.
// toolchain_mappings: [
//     {prefix: "myapp", toolchain: "shell"},
// ]
`, name)
}
```

### 2. Update verbose output

Update the success message at the end of `Init()` to mention the new files:

```go
if opts.Verbose {
    fmt.Printf("Created %s\n", pudlDir)
    fmt.Printf("  workspace.cue  (workspace: %q)\n", dirName)
    fmt.Printf("  schema/        (project-specific CUE schemas)\n")
    fmt.Printf("  definitions/   (desired state definitions)\n")
}
```

### 3. Ensure .gitignore awareness

The workspace files should be git-tracked (they're desired state). But add a note to the repo's `.gitignore` suggestion:

```
# .pudl/ contents are git-tracked (definitions, schemas, workspace config)
# Do NOT add .pudl/ to .gitignore
```

## Acceptance Criteria

- `pudl repo init` creates `workspace.cue` with the directory name as workspace name
- `pudl repo init` creates `schema/` and `definitions/` subdirectories
- `workspace.cue` is valid CUE and parseable by `workspace.Discover()`
- Running `pudl repo init` twice without `--force` does not overwrite an existing `workspace.cue`
- Running with `--force` regenerates `workspace.cue`
