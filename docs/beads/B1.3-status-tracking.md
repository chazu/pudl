# B1.3 — Convergence Status Tracking

**Epic:** E1 — Close the ACUTE Feedback Loop
**Blocked by:** B1.1 (observe ingestion), B1.2 (manifest ingestion)
**Blocks:** B1.4 (status CLI)

## Summary

Add a `status` column to the catalog and update it throughout the ACUTE cycle so each resource has a queryable convergence state.

## Background

- The catalog has `resource_id` for logical identity and `version` for change tracking
- No field currently tracks convergence state
- Status must be updated by drift check, export-actions, manifest ingest, and observe ingest
- Database migrations in pudl are idempotent — they check for column existence before altering

## Implementation

### 1. Modify: `internal/database/catalog.go` — Migration

Add a new migration function `ensureStatusColumn()` called from `NewCatalogDB`:

```go
func (db *CatalogDB) ensureStatusColumn() error {
    // Check if column exists
    var count int
    err := db.db.QueryRow(
        "SELECT COUNT(*) FROM pragma_table_info('catalog_entries') WHERE name='status'",
    ).Scan(&count)
    if err != nil {
        return err
    }
    if count > 0 {
        return nil // Already exists
    }

    _, err = db.db.Exec("ALTER TABLE catalog_entries ADD COLUMN status TEXT DEFAULT 'unknown'")
    return err
}
```

Register in `NewCatalogDB` alongside existing migrations (`ensureIdentityColumns`, `ensureArtifactColumns`).

### 2. Modify: `internal/database/catalog.go` — Status Update

Add method:

```go
// UpdateStatus sets the convergence status for entries matching a definition name.
// Only updates the latest entry for the definition.
func (db *CatalogDB) UpdateStatus(definitionName string, status string) error {
    validStatuses := map[string]bool{
        "unknown": true, "clean": true, "drifted": true,
        "converging": true, "converged": true, "failed": true,
    }
    if !validStatuses[status] {
        return fmt.Errorf("invalid status: %s", status)
    }
    _, err := db.db.Exec(
        `UPDATE catalog_entries SET status = ?, updated_at = CURRENT_TIMESTAMP
         WHERE definition = ? AND id = (
             SELECT id FROM catalog_entries WHERE definition = ?
             ORDER BY import_timestamp DESC LIMIT 1
         )`,
        status, definitionName, definitionName,
    )
    return err
}

// GetDefinitionStatuses returns the latest status for each definition that has entries.
func (db *CatalogDB) GetDefinitionStatuses() ([]DefinitionStatus, error)
```

Where `DefinitionStatus` is:

```go
type DefinitionStatus struct {
    Definition string
    Status     string
    UpdatedAt  time.Time
    DiffCount  int // from latest drift report, 0 if clean
}
```

SQL for `GetDefinitionStatuses`:
```sql
SELECT definition, status, updated_at
FROM catalog_entries
WHERE definition IS NOT NULL AND definition != ''
GROUP BY definition
HAVING import_timestamp = MAX(import_timestamp)
ORDER BY definition
```

## Status State Machine

```
unknown ──drift check──> clean
unknown ──drift check──> drifted
drifted ──export-actions──> converging
converging ──ingest-manifest (exit 0)──> converged
converging ──ingest-manifest (exit != 0)──> failed
converged ──drift check──> clean
converged ──drift check──> drifted (something changed externally)
failed ──drift check──> drifted
clean ──drift check──> drifted (something changed externally)
```

Any state can transition to any other via the appropriate operation. There is no enforced state machine — status is always set to the latest known truth.

## Acceptance Criteria

- New and existing catalog entries have `status` column defaulting to `"unknown"`
- `pudl drift check my_app` sets status to `"clean"` or `"drifted"`
- `pudl export-actions --definition my_app` sets status to `"converging"`
- `pudl ingest-manifest` sets status to `"converged"` or `"failed"` per action exit code
- `GetDefinitionStatuses()` returns correct per-definition status
