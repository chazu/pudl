# B1.2 — Manifest Ingestion

**Date:** 2026-03-24
**Bead:** B1.2 (Epic 1: Close the ACUTE Feedback Loop)

## Summary

Implemented `pudl ingest-manifest` command that reads mu build manifest JSON and records per-action results in the catalog. This enables pudl to know what mu did and whether it succeeded.

## Files Created

- `internal/mubridge/manifest.go` — Core ingestion logic
- `internal/mubridge/manifest_test.go` — Tests (5 test functions)
- `internal/database/catalog_manifest.go` — GetManifestActions and GetLatestManifestAction query methods
- `cmd/ingest_manifest.go` — CLI command

## Public API

### `mubridge.IngestManifest(db, reader, origin, configDir) (*IngestManifestResult, error)`
Reads manifest JSON, creates one `manifest` catalog entry and one `manifest-action` entry per action. Content-hash dedup skips duplicate manifests.

### Types
- `ManifestInput` — Top-level manifest with timestamp, summary, actions
- `ManifestSummary` — Aggregate counts (total, cached, executed, failed)
- `ManifestAction` — Per-action result (id, target, cached, exit_code, outputs)
- `IngestManifestResult` — Ingestion result (run_id, totals, skipped flag)

### Database Methods
- `CatalogDB.GetManifestActions(runID) ([]CatalogEntry, error)` — All manifest-action entries for a run
- `CatalogDB.GetLatestManifestAction(definitionName) (*CatalogEntry, error)` — Most recent action for a definition

### CLI
- `pudl ingest-manifest --path <file> --origin <string>` — Ingest manifest from file or stdin

## Entry Types
- `manifest` — Schema `pudl/mu.#Manifest`, contains full manifest JSON
- `manifest-action` — Schema `pudl/mu.#ManifestAction`, one per action with tags `{"exit_code": N, "cached": bool}`
