# B1.4 — Status CLI Command

**Date:** 2026-03-25
**Bead:** B1.4

## Summary

Implemented `pudl status [definition]` command that displays per-definition convergence state.

## Public API

### CLI

- `pudl status` — table view of all definitions with status, last updated, and diff count
- `pudl status <name>` — detailed view of a single definition including drift differences
- `pudl status --json` — machine-readable JSON output (works with both modes)

### Status color coding (terminal)

- Green: `clean`, `converged`
- Yellow: `drifted`, `converging`
- Red: `failed`
- Gray: `unknown`

## Files Created

- `cmd/status.go` (~210 lines) — command implementation with table/JSON/detail output
- `cmd/status_test.go` (~100 lines) — unit tests for helpers and output structs

## Files Modified

- `docs/plan.md` — marked B1.4 as complete

## Implementation Details

- Uses `database.GetDefinitionStatuses()` from B1.3 for the table view
- Enriches drifted definitions with diff counts from `drift.ReportStore.GetLatest()`
- Single-definition detail view loads the full drift report and prints field-level differences
- Color coding via lipgloss (already a project dependency)
- JSON output uses `StatusOutput` and `StatusDetailOutput` structs with `omitempty` for optional fields
- Follows existing patterns from `cmd/drift_check.go` for DB/config initialization
