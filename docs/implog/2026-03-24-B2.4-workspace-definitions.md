# B2.4 — Workspace-Aware Definition Discovery

**Date:** 2026-03-24
**Bead:** B2.4

## Summary

Extended the definition discoverer to search multiple schema paths in order, enabling per-repo definitions to shadow global definitions with the same name.

## Changes

### `internal/definition/discovery.go`
- Changed `Discoverer` struct from single `schemaPath string` to `searchPaths []string`
- `NewDiscoverer(schemaPath)` still works identically (creates single-element search path)
- Added `NewMultiDiscoverer(schemaPaths []string)` for multi-path discovery
- Replaced `definitionsDir()` with `definitionsDirs()` returning all definition directories
- Updated `ListDefinitions()` to iterate all dirs with a `seen` map so first-found wins (shadowing)

### `internal/definition/definition_test.go`
- Added `TestMultiDiscoverer_PerRepoFirst` — per-repo definitions shadow global ones
- Added `TestMultiDiscoverer_MergesBoth` — non-conflicting definitions from both paths are merged
- Added `TestMultiDiscoverer_EmptyPerRepo` — empty per-repo dir falls through to global
- Added `TestMultiDiscoverer_NoDirs` — all missing paths returns empty list
- Added `TestNewDiscoverer_BackwardCompat` — single-path constructor works identically

## Public API

```go
// Existing (unchanged signature, backward compatible):
func NewDiscoverer(schemaPath string) *Discoverer

// New:
func NewMultiDiscoverer(schemaPaths []string) *Discoverer
```

## Notes

- `cmd/` wiring (using workspace context to construct multi-discoverers in commands) is handled by B2.3
- Pre-existing build failures in database/importer packages are unrelated to this change
