# pudl repo init command

## Summary
Added `pudl repo init` subcommand for initializing PUDL in a specific repository. This creates a `.pudl/` directory and installs Claude skills into `.claude/skills/`.

## Files Changed
- `internal/repo/init.go` — new package with `Init()` function
- `internal/repo/init_test.go` — tests for init, already-exists, and force-overwrite
- `cmd/repo.go` — added `repoInitCmd` subcommand with `--force` flag

## Public API
- `repo.Init(opts InitOptions) error` — initialize a repo with `.pudl/` and Claude skills
- `repo.InitOptions{Dir, Force, Verbose}` — configuration for repo init
- CLI: `pudl repo init [--force]`

## Design Decisions
- `.pudl/` is minimal for now (just a directory marker), establishing the convention for future project-local models/definitions/methods
- `.claude/` and `.claude/skills/` are created if they don't exist
- Reuses existing `skills.WriteSkills()` for skill installation
- Errors if `.pudl/` already exists unless `--force` is passed
